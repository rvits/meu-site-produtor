## Resumo da versão

A v1.0.0 consolida o site operacional da THouse Rec para clientes reais: registro, agendamento, pagamento via Asaas, painel Minha Conta, administração e fluxos de reembolso. Esta release encerra o ciclo de hardening (HS, PH, RC, GO) sem abrir o StudioOS nem novas funcionalidades de produto.

A branch candidata contém **16 commits** à frente de `main`, **33 migrations** Prisma e pipeline completo de certificação (RC-01…04, LAUNCH-01, GO-01/02).

## Arquitetura consolidada

- **Stack:** Next.js 16 (App Router), Prisma 5, PostgreSQL (Neon), Vercel
- **Pagamento:** Asaas (produção) — webhook em `/api/webhooks/asaas`
- **Auth:** Cookie `session_id`, sessão 7 dias, role `ADMIN`
- **Domínio:** Appointment → Service → Payment → Coupon (ver `docs/architecture/v1.0-overview.md`)
- **Engines:** Workflow SM (HS-03B), SYNC-01A, SIM-01/02, EC-01, TE-01B/02A
- **StudioOS:** Congelado — não incluído nesta release

## Limitações

- Carrinho apenas no browser (sessionStorage) — perdido ao trocar dispositivo
- Entrega presencial no estúdio — sem tracking digital de "serviço entregue"
- Reembolso reativo (após cancelamento admin)
- Execution Core / TE / SIM bloqueados em Production (por design EC-01)
- Rollback de schema via Neon PITR — Prisma não suporta `migrate down` automático
- CPF `@unique` — migration HS-01 falha se houver duplicatas no banco

## Planejamento inicial da v1.1

**Escopo sugerido (pós-estabilização 2–4 semanas):**

1. **H1 hardening (E2E-02)** — Polling pós-pagamento, redirect `/minha-conta`, mensagens Asaas
2. **Webhook resiliência** — Logging estruturado, alertas em falha de efeitos
3. **UX operacional** — SLA visível ao cliente, card reembolso proativo
4. **Cupom PR-04** — Ownership e remoção de shims legados
5. **RC03-RACE-01** — Lock otimista ou validação atômica de slot

**Critérios para iniciar v1.1:** 7 dias sem ticket crítico de pagamento + webhook OK + ≥5 agendamentos reais concluídos.
