## Fluxo de pagamento

```
Cliente autenticado
    │
    ├─► Carrinho (/carrinho) ──► POST /api/asaas/checkout-carrinho
    │         │                        │
    │         │                        ├─► PaymentMetadata (payload checkout)
    │         │                        └─► Asaas cobrança + redirect URL
    │
    ├─► Agendamento direto ──► POST /api/asaas/checkout-agendamento
    │
    └─► Plano ──► POST /api/asaas/checkout
              │
              ▼
        Cliente paga no Asaas
              │
              ▼
        POST /api/webhooks/asaas  (PAYMENT_RECEIVED)
              │
              ├─► process-payment-webhook.ts
              ├─► Payment (approved) + Appointment / UserPlan / Coupons
              └─► asaas-*-payment-effects.ts
              │
              ▼
        Minha Conta reflete agendamento/pagamento
```

**Pontos críticos:** `ASAAS_WEBHOOK_ACCESS_TOKEN`, `NEXT_PUBLIC_SITE_URL`, idempotência por `asaasId`.

## Fluxo de agendamento

```
/registro (opcional) ──► /agendamento
    │
    ├─► Seleção serviços/beats + data/hora
    ├─► GET disponibilidade + blocked-slots
    ├─► Carrinho (localStorage) ou cupom 100%
    │
    ├─► POST /api/agendamentos (criação direta)
    ├─► POST /api/agendamentos/com-cupom
    └─► Checkout Asaas (ver fluxo pagamento)
              │
              ▼
        Appointment (pendente) ──► Admin aceita/recusa
              │
              ▼
        Service vinculados ──► status confirmado/concluído
```

**State Machine:** `pendente → aceito → confirmado → em_andamento → concluido` (ou `recusado` / `cancelado` / `remarcado`).

## Workflow & State Machine (HS-03B)

| Componente | Localização |
|------------|-------------|
| Transições canônicas | `src/app/lib/workflow/` |
| Histórico persistido | `DomainTransitionHistory` (Prisma) |
| Auditoria | `npm run workflow:audit` |

Toda transição oficial deve passar pela SM e registrar `eventName`, `actorType`, `fromStatus`, `toStatus`.

## Execution Core (EC-01)

| Papel | Descrição |
|-------|-----------|
| Runner | `src/app/lib/execution/runner.ts` |
| Discovery | Registro de cenários TE/SIM/RC |
| Permissões | Bloqueado em Production; ADMIN ou CLI secret em dev/preview |
| API | `POST /api/execution/run` |

**Scripts:** `npm run te:run`, `npm run sim:run`, `npm run exec:audit`

---

## Synchronization (SYNC-01A)

| Papel | Descrição |
|-------|-----------|
| Outbox | Tabela `SynchronizationEvent` (cursor monotônico) |
| Engine | `src/app/lib/synchronization/` |
| Cliente | `DomainSyncProvider` + SSE replay por cursor |
| Escopos | `user`, `admin`, `public`, `all` |

**Script:** `npm run sync:audit`
